# FinFlow Backend API

Полноценный бэкенд для no-code платформы FinFlow, построенный на Node.js, Express и TypeScript.

## Функциональность

### ✨ Основные возможности

- **Аутентификация и авторизация** - JWT токены, защищенные маршруты
- **Управление воркфлоу** - CRUD операции для создания и редактирования процессов
- **Выполнение воркфлоу** - Движок для обработки документов по заданным правилам
- **Загрузка файлов** - Поддержка PDF и TXT файлов
- **Дашборд и отчетность** - Статистика и история выполнений
- **База данных** - SQLite с автоматической инициализацией таблиц

### 🔧 Технологический стек

- **Node.js** - Среда выполнения
- **Express.js** - Веб-фреймворк
- **TypeScript** - Типизация
- **SQLite** + **Knex.js** - База данных и ORM
- **JWT** - Аутентификация
- **Multer** - Загрузка файлов
- **PDF-Parse** - Обработка PDF
- **Helmet** - Безопасность
- **CORS** - Кросс-доменные запросы

## 📋 API Endpoints

### Аутентификация (`/api/auth`)
- `POST /register` - Регистрация пользователя
- `POST /login` - Вход в систему
- `GET /me` - Информация о текущем пользователе
- `POST /logout` - Выход из системы

### Воркфлоу (`/api/workflows`)
- `GET /` - Получить список воркфлоу
- `POST /` - Создать новый воркфлоу
- `GET /:id` - Получить воркфлоу по ID
- `PUT /:id` - Обновить воркфлоу
- `DELETE /:id` - Удалить воркфлоу

### Выполнение (`/api/executions`)
- `POST /:workflowId` - Выполнить воркфлоу (с файлом или текстом)
- `POST /:workflowId/text` - Выполнить воркфлоу только с текстом

### Дашборд (`/api/dashboard`)
- `GET /stats` - Получить статистику
- `GET /reports` - Получить отчеты

## 🚀 Установка и запуск

### Локальная разработка

1. **Установка зависимостей:**
   ```bash
   cd backend
   npm install
   ```

2. **Настройка переменных окружения:**
   ```bash
   cp .env.example .env
   # Отредактируйте .env файл по необходимости
   ```

3. **Инициализация базы данных:**
   ```bash
   npm run db:seed
   ```

4. **Запуск сервера разработки:**
   ```bash
   npm run dev
   ```

5. **Сборка для продакшена:**
   ```bash
   npm run build
   npm start
   ```

### Docker

```bash
# Из корневой директории проекта
docker-compose up --build
```

## 🔧 Workflow Engine

Движок поддерживает следующие типы узлов:

### Входные
- `START` - Начальный узел
- `UPLOAD` - Загрузка документа

### Извлечение данных
- `EXTRACT_DATA` - Извлечение по регулярному выражению
- `EXTRACT_TABLE` - Извлечение табличных данных
- `EXTRACT_MULTIPLE` - Извлечение всех совпадений

### Обработка данных
- `MATH_OPERATION` - Математические операции
- `FORMAT_NUMBER` - Форматирование чисел
- `FORMAT_DATE` - Форматирование дат
- `TEXT_REPLACE` - Замена текста
- `TEXT_JOIN` - Объединение текста
- `SET_VARIABLE` - Установка переменной

### Логика и контроль
- `COMPARE_VALUES` - Сравнение значений
- `CONDITION` - Условное ветвление
- `SWITCH` - Множественный выбор
- `CHECK_EXISTS` - Проверка существования
- `VALIDATE_PATTERN` - Валидация по шаблону

### Вывод
- `OUTPUT_SUCCESS` - Успешное завершение
- `OUTPUT_ERROR` - Завершение с ошибкой
- `LOG_MESSAGE` - Запись в лог

## 🤝 Разработка

### Добавление новых типов узлов

1. Добавьте тип в `NodeType` enum в `types/index.ts`
2. Создайте интерфейс настроек
3. Реализуйте логику в `WorkflowEngine.executeNode()`
4. Добавьте валидацию в контроллеры

### Расширение API

1. Добавьте новые маршруты в папку `routes/`
2. Создайте контроллер в `controllers/`
3. Добавьте валидацию в `middleware/validation.ts`
4. Обновите типы в `types/index.ts`
